# Project Goal
Build a production-ready Flutter personal finance tracker app for Android and iOS.

# Stack
- Flutter
- Dart
- Offline-first
- Local storage/database
- Indonesian as default language

# Core Features
- Income tracking
- Expense tracking
- Daily summary
- Monthly summary
- Yearly summary
- Date-range summary
- Payday-cycle summary
- Daily reminder notification at 9 PM
- Finance quotes on home screen
- Modern and user-friendly UI

# Non-Negotiable Requirements
- App must be runnable
- No pseudocode
- No TODO placeholders
- No fake stubs
- No broken imports
- No unresolved dependencies
- Must create:
  - docs/DEVELOPMENT_TO_DEPLOY.md
  - docs/CODEBASE_HANDOVER.md
- Must include enough assets so app looks complete

# Engineering Rules
- Clean and maintainable architecture
- Reusable widgets/components
- Clear separation of layers
- Good state management
- Accurate financial calculations
- Add tests for business logic
- Keep files readable
- Avoid giant files

# UX Rules
- Clean modern premium UI
- Good form validation
- Good loading/empty/error states
- Friendly for non-technical users

# Documentation Rules
- Explain architecture
- Explain folders and major files
- Explain report logic
- Explain payday-cycle logic
- Explain notification logic
- Explain how future developers continue the project

# Workflow
1. Plan first
2. Implement step by step
3. Verify thoroughly
4. Fix discovered issues
5. Complete documentation
6. Final cleanup
